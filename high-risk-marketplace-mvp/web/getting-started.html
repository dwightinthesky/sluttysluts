Getting Started

Enterprise setup for Landing + Console + Backend flow

Landing Open Console

Production-style local bring-up in under 5 minutes

Run backend APIs, verify end-to-end payment lifecycle, and then open role-based workspaces from the Landing page.

1) Boot backend services

Install dependencies and start the API server.

cd ../backend
npm install
npm start

2) Run full smoke validation

Execute the scripted journey: KYC, listing, order, webhook, payout, wallet, chat.

# new terminal
cd ../backend
npm run smoke

3) Open SaaS interfaces

  • Landing: role entry + language selector
  • Console: customer, creator, operations, API logs
  • Set API base to match your local backend

Reference architecture (MVP)

Landing + Console UI GitHub Pages static frontend
Backend API Layer Order, promotion, wallet, webhook orchestration
Data and ledger model Orders, payments, reserve, payouts, audit events
Tip: enter via Landing role buttons so role/lang is passed into the console URL.
