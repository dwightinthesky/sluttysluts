# PRD（MVP）

## 1. 問題定義
高風險成人利基市場在需求端存在強隱私、高客單與高復購特性，但平台常因金流封鎖、法遵失誤、物流隱私外洩與退單失控而快速失敗。

## 2. 產品目標
- 在合規前提下，提供「比私下交易更安全」的交易體驗。
- 透過風險分層與自動化審核，降低人工作業成本與法務暴露。
- 建立可被審計的交易生命週期（Auditability）。

## 3. 角色
- 買家（Buyer）：已成年、重視隱私、期望可驗證履約。
- 創作者/賣家（Seller）：已成年、需穩定收款與風險保護。
- 風控與審核員（Ops/Trust）：處理爭議、審核、封鎖。
- 平台管理員（Admin）：政策管理、供應商管理、報表監控。

## 4. MVP 範圍
### 4.1 功能需求
- 帳號與身分
  - Email/OTP 註冊
  - 年齡驗證（買家強制）
  - KYC/KYB（賣家強制）
- 商品與內容
  - 商品上架、價格、庫存、標籤
  - 圖文內容預審（AI + 人工覆核）
- 下單與支付
  - 單筆交易下單
  - 支付編排層（主備 PSP）
  - 交易狀態機（pending/authorized/paid/held/released/refunded）
- 履約與物流
  - 物流狀態回傳 webhook
  - 到貨確認後觸發可撥款
- 風控與爭議
  - 聊天外流偵測（外部聯絡方式/跳轉金流）
  - 申訴/爭議建立、證據上傳、裁決
- 撥款
  - 平台佣金拆分
  - 賣家可提領餘額

### 4.2 非功能需求
- 資安：傳輸 TLS、靜態加密、最小權限。
- 隱私：最小化蒐集、資料保留期限、刪除流程。
- 可觀測性：審計事件、錯誤告警、異常交易報表。
- 韌性：PSP 故障可切換，訂單資料不可遺失。

## 5. 明確排除
- 不處理未成年相關內容與交易。
- 不允許違法內容、非自願內容、暴力或剝削內容。
- 不允許違反承運商政策的包裹內容與申報方式。

## 6. 主要流程
1. 賣家註冊 -> KYC 通過 -> 開店。
2. 賣家上架 -> 審核通過 -> 可被購買。
3. 買家完成年齡驗證 -> 下單支付。
4. 訂單付款成功 -> 進入履約監控。
5. 到貨確認/爭議期結束 -> 自動撥款。

## 7. KPI
- 交易層
  - 授權成功率、支付成功率、退單率
- 合規層
  - 驗證失敗率、可疑交易告警率、人工覆核 SLA
- 成長層
  - 賣家留存、買家復購率、ARPPU

## 8. 驗收標準（MVP）
- 可完成完整 happy path：註冊到撥款。
- 每個關鍵事件有審計紀錄。
- 失敗流程可回復（例如 PSP 超時重試）。
